# Task: iOS Sentry SDK Integration

## Description
Integrate the Sentry-Swift SDK into the TrackRat iOS project to enable comprehensive error monitoring, session replay, performance tracing, and profiling. This involves adding the SDK dependency, initializing it in the main app entry point, and configuring environment-specific settings for staging and production deployments.

## Acceptance Criteria
- [ ] Sentry-Swift SDK added to iOS project via Package.swift or Podfile
- [ ] SDK initialized in TrackRatApp.swift with full feature set enabled
- [ ] Error monitoring configured and functional
- [ ] Session replay enabled with user consent mechanism
- [ ] Performance tracing and profiling configured
- [ ] Environment-based configuration (staging vs production) implemented
- [ ] Appropriate sampling rates configured per environment
- [ ] User consent flow for session replay implemented
- [ ] DSN and configuration stored securely

## Technical Details
- Add Sentry-Swift SDK as dependency using Swift Package Manager or CocoaPods
- Initialize SDK in TrackRatApp.swift's init() or App.onAppear
- Configure different DSNs for staging/production environments
- Set up sampling rates: 100% for staging, optimized rates for production
- Implement user consent UI for session replay feature
- Configure Info.plist with necessary privacy descriptions
- Ensure proper error boundaries and crash reporting

## Dependencies
- None (can run in parallel with backend tasks)
