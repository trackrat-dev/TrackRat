# Task: Add Backend Custom Instrumentation

## Description
Add comprehensive custom instrumentation throughout the backend to provide detailed performance monitoring and error context. This includes performance spans for transit API calls, custom error types with business context, database query instrumentation, and custom tags for train IDs, stations, and data sources.

## Acceptance Criteria
- [ ] Add performance spans for transit API calls in all collectors
- [ ] Create custom error types with business context for common failure scenarios
- [ ] Instrument database queries with performance monitoring
- [ ] Add custom tags for train IDs, stations, and data sources to all relevant operations
- [ ] Add user context for API requests
- [ ] Implement transaction naming for better grouping in Sentry
- [ ] Add breadcrumbs for key operations (API calls, data processing)

## Technical Details
This task involves adding detailed instrumentation across the backend:

Performance spans:
- Wrap all external API calls in collectors with performance spans
- Add spans for data processing operations
- Instrument database operations with custom spans

Custom error handling:
- Create domain-specific error classes (TrainDataError, StationLookupError, etc.)
- Add business context to errors (which train, station, time range)
- Implement proper error capture with relevant tags

Files to modify:
- backend_v2/src/trackrat/collectors/njt/client.py - add spans and error context
- backend_v2/src/trackrat/collectors/amtrak/client.py - add spans and error context
- backend_v2/src/trackrat/services/departure.py - add performance monitoring
- backend_v2/src/trackrat/api/trains.py - add user context and transaction naming
